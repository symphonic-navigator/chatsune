Design Overview -- 5 Arbeitspakete

Zusammenfassung der besprochenen Aenderungen am Prototyp

1

Auto-detect Master Admin

Backend: GET /api/auth/status
Frontend: LoginPage erkennt Setup-Modus automatisch

2

Toast Notification System

Zustand-Store + Toast-Komponente
Ersetzt alle alert() Aufrufe
Bell/Flyout → FOR_LATER.md

3

ModelBrowser mit Filtern

Suche + Provider + Capabilities + Favourites
Ersetzt die einfache Tabelle auf der Models-Seite
Wiederverwendbare Komponente

4

Admin: System Prompt Tab

Globaler System-Prompt auf der AdminPage
Wird bei jeder Inference vorangestellt
Zentrales Sicherheits-Feature

5

Admin: API Key Status pro User

Neuer LLM-Endpoint: /api/llm/admin/credential-status
UsersPage merged Daten im Frontend
Module bleiben sauber getrennt
